Backend Strategy

Informe Técnico del Backend de HirePath

Guía para construir el backend en Django y Django REST Framework, explicando el propósito de cada tabla, su relación con los workflows del sistema y la lógica operativa de búsqueda de empleo, CV dinámico, recomendación salarial, postulación asistida y preparación de entrevistas.

1. Objetivo del backend

El backend de HirePath debe funcionar como el núcleo de una plataforma de empleabilidad inteligente. Su responsabilidad principal es almacenar el perfil profesional real del usuario, importar o capturar vacantes desde fuentes externas, analizar compatibilidad, generar CVs adaptados, sugerir salarios, preparar respuestas de postulación y construir discursos para entrevistas.

El sistema no está pensado para empresas usuarias. Las empresas solo aparecen como información externa asociada a vacantes scrapeadas o importadas desde portales laborales.

2. Consideración importante sobre scraping y automatización

Aunque el sistema puede tener módulos para capturar información de portales laborales, se recomienda priorizar APIs oficiales, integraciones permitidas, feeds públicos o carga manual/asistida de URLs. La automatización sobre sitios como LinkedIn, Computrabajo o Bumeran debe revisarse legal y técnicamente, porque algunos portales restringen bots, scraping o postulación automática.

La estrategia más segura para el MVP es trabajar con un modelo de postulación asistida: el sistema prepara la información, pero el usuario revisa y aprueba antes de enviar.

Capa Tecnología Responsabilidad

5. Propósito de cada tabla

5.1 CandidateProfile

Representa el perfil profesional general del usuario. Guarda información resumida como titular profesional, resumen, años de experiencia, salario actual, rango salarial deseado, ubicación y enlaces profesionales.

Workflow relacionado: creación del perfil, matching con vacantes, generación de CV, recomendación salarial y preparación de entrevista.

5.2 Education

Guarda la formación académica real del usuario: institución, grado, carrera, fechas y descripción. Esta información puede mostrarse o excluirse de un CV dependiendo de la vacante.

Workflow relacionado: CV dinámico y respuestas de postulación.

5.3 WorkExperience

Almacena la experiencia laboral del usuario. Es una de las tablas más importantes porque alimenta el análisis de compatibilidad, el CV adaptado y las respuestas de entrevista.

Workflow relacionado: matching, CV dinámico, speech de entrevista y respuestas automáticas a formularios.

5.4 Project

Guarda proyectos personales, académicos o laborales. Es útil para candidatos que tienen experiencia práctica demostrable, incluso si no proviene de empleos formales.

Workflow relacionado: CV dinámico, entrevistas técnicas y diferenciación del candidato.

5.5 Certification

Guarda certificados, cursos, credenciales y enlaces de validación. Permite reforzar la candidatura cuando una vacante pide herramientas, metodologías o conocimientos específicos.

Workflow relacionado: CV dinámico y matching por requisitos.

5.6 Skill y UserSkill

Skill funciona como catálogo maestro de habilidades. UserSkill relaciona cada habilidad con un usuario, indicando nivel y años de experiencia.

Workflow relacionado: cálculo de compatibilidad, filtros de vacantes, CV dinámico y respuestas técnicas.

5.7 JobSource

Representa las fuentes externas desde donde se obtienen vacantes: LinkedIn, Computrabajo, Bumeran, Indeed, páginas corporativas u otras fuentes.

Workflow relacionado: scraping, auditoría de fuentes y control de qué portales están activos.

5.8 ExternalCompany

Guarda empresas detectadas desde las vacantes. No son cuentas usuarias dentro de HirePath; solo son entidades externas usadas para clasificar oportunidades.

Workflow relacionado: organización de vacantes, historial de postulaciones y preparación de entrevista por empresa.

5.9 JobPosting

Guarda cada vacante encontrada o importada. Contiene título, descripción, requisitos, responsabilidades, ubicación, modalidad, salario publicado, URL, fuente y estado.

Workflow relacionado: búsqueda de empleo, matching, generación de CV, postulación y entrevista.

5.10 JobSearchCriteria

Guarda los criterios configurados por el usuario: cargos objetivo, palabras clave, palabras excluidas, ubicaciones, modalidad, seniority y salario mínimo.

Workflow relacionado: búsqueda personalizada y selección de vacantes compatibles.

5.11 JobMatch

Guarda el resultado del análisis entre un usuario y una vacante. Incluye score, habilidades coincidentes, habilidades faltantes, experiencia relacionada, riesgos y explicación generada por IA.

Workflow relacionado: decisión de postular o descartar una vacante.

5.12 SalaryBenchmark

Guarda datos de mercado salarial por rol, ciudad, país y seniority. Puede alimentarse manualmente, por investigación o por fuentes externas permitidas.

Workflow relacionado: recomendación salarial.

5.13 SalaryRecommendation

Guarda el salario sugerido para una vacante específica y un usuario específico. Considera experiencia, seniority, salario publicado, mercado y expectativas del usuario.

Workflow relacionado: formularios de postulación y preparación para preguntas de expectativa salarial.

5.14 CVTemplate

Guarda las plantillas HTML y CSS que se usarán para construir CVs en PDF. Permite tener varios estilos de CV según tipo de puesto o preferencia del usuario.

Workflow relacionado: generación de CV dinámico.

5.15 GeneratedCV

Guarda cada CV generado para una vacante. Registra qué educación, experiencia, proyectos, certificaciones y habilidades fueron seleccionadas.

Workflow relacionado: postulación y trazabilidad del contenido enviado.

5.16 Application

Representa la postulación del usuario a una vacante. Conecta usuario, vacante, CV generado, salario recomendado, estado de avance, URL y notas.

Workflow relacionado: CRM laboral o pipeline de postulaciones.

5.17 ApplicationQuestion

Guarda preguntas encontradas en formularios de postulación y sus respuestas sugeridas o finales. Permite que la IA proponga respuestas y el usuario las revise.

Workflow relacionado: postulación asistida.

5.18 InterviewPrep

Guarda una preparación de entrevista asociada a un usuario, vacante y postulación. Puede incluir recomendaciones generales para esa entrevista.

Workflow relacionado: preparación para entrevista.

5.19 InterviewQuestion

Guarda preguntas probables de entrevista y respuestas sugeridas. Se pueden clasificar como generales, técnicas, conductuales, salariales o de afinidad con la empresa.

Workflow relacionado: speech personalizado para entrevistas.

5.20 ScrapingRun

Registra cada ejecución de scraping por fuente. Guarda estado, hora de inicio, fin, cantidad de vacantes encontradas, creadas, actualizadas y errores.

Workflow relacionado: monitoreo del scraping.

5.21 ScrapedPageLog

Guarda el detalle de páginas visitadas durante una corrida de scraping, incluyendo URL, código de estado, éxito o error.

Workflow relacionado: auditoría y depuración del scraping.

5.22 AIRequestLog

Guarda cada solicitud hecha a Groq u otro proveedor de IA. Registra tarea, input, output, modelo, proveedor y tokens usados.

Workflow relacionado: auditoría de IA, control de costos y trazabilidad de decisiones.

6. Workflows principales del sistema

6.1 Workflow de registro del candidato

  1. El usuario crea una cuenta.
  2. Completa su CandidateProfile.
  3. Registra educación, experiencia, proyectos, certificaciones y habilidades.
  4. El sistema valida que la información sea real, ordenada y reutilizable.
  5. El perfil queda disponible para matching, CV dinámico y entrevistas.

6.2 Workflow de búsqueda de empleo

  1. El usuario configura JobSearchCriteria.
  2. El sistema consulta fuentes activas desde JobSource.
  3. Se ejecuta un ScrapingRun.
  4. Por cada página visitada se registra un ScrapedPageLog.
  5. Las empresas encontradas se guardan como ExternalCompany.
  6. Las vacantes se guardan como JobPosting.

6.3 Workflow de matching

  1. El sistema toma un JobPosting activo.
  2. Compara requisitos contra experiencia, proyectos, certificaciones y skills del usuario.
  3. Groq puede generar una explicación del nivel de compatibilidad.
  4. El resultado se guarda en JobMatch.
  5. Si el score es adecuado, la vacante pasa a recomendada.

6.4 Workflow de recomendación salarial

  1. El sistema revisa salario publicado en JobPosting, si existe.
  2. Consulta referencias internas desde SalaryBenchmark.
  3. Considera seniority, ubicación, experiencia del usuario y expectativa mínima.
  4. Genera una recomendación con Groq o reglas internas.
  5. Guarda el resultado en SalaryRecommendation.

6.5 Workflow de CV dinámico

  1. El usuario selecciona una vacante recomendada.
  2. El sistema identifica información relevante del perfil.
  3. Groq redacta un resumen profesional adaptado sin inventar datos.
  4. Se seleccionan experiencias, proyectos, certificaciones y habilidades.
  5. Se genera un GeneratedCV usando una CVTemplate.
  6. El PDF queda disponible para revisión o postulación.

6.6 Workflow de postulación asistida

  1. Se crea una Application en estado PENDING_REVIEW.
  2. El sistema sugiere respuestas para preguntas del formulario.
  3. Las preguntas se guardan en ApplicationQuestion.
  4. El usuario revisa el CV, salario sugerido y respuestas.
  5. Si aprueba, la postulación pasa a estado APPLIED.

6.7 Workflow de entrevista

  1. Cuando una postulación cambia a INTERVIEW, se crea un InterviewPrep.
  2. Groq analiza la vacante y el perfil del usuario.
  3. Genera preguntas probables y respuestas sugeridas.
  4. Las respuestas se guardan en InterviewQuestion.
  5. El usuario puede practicar y agregar notas propias.

8. Servicios internos recomendados

Para evitar que los ViewSets tengan demasiada lógica, se recomienda usar servicios separados dentro del backend.

Servicio Responsabilidad
JobScrapingService Buscar vacantes en fuentes externas y guardar resultados.
JobMatchingService Comparar perfil del usuario contra una vacante.
SalaryRecommendationService Calcular salario sugerido para una vacante.
CVGenerationService Seleccionar contenido relevante y generar CV en HTML/PDF.
ApplicationAnswerService Responder preguntas de formularios de postulación.
InterviewPrepService Crear preguntas y respuestas para entrevistas.
GroqClient Centralizar todas las llamadas al LLM.

9. Reglas de negocio importantes

  • Un usuario solo puede ver y modificar su propia información.
  • No se debe generar información falsa en el CV.
  • Toda respuesta generada por IA debe poder ser revisada por el usuario.
  • Una vacante no debe duplicarse si ya existe por url o external_job_id.
  • Una postulación debe ser única por usuario y vacante.
  • El salario sugerido debe indicar nivel de confianza y razonamiento.
  • Los logs de IA deben guardarse para trazabilidad.
  • El scraping debe registrar errores y resultados por fuente.

10. Orden recomendado de desarrollo

  1. Crear proyecto Django y configurar PostgreSQL.
  2. Implementar autenticación JWT.
  3. Crear modelos, migraciones y Django Admin.
  4. Crear serializers y ViewSets para perfil profesional.
  5. Crear endpoints de vacantes y criterios de búsqueda.
  6. Crear carga manual de vacantes antes del scraping.
  7. Implementar matching básico con reglas.
  8. Integrar Groq para mejorar matching y explicación.
  9. Crear generación de CV HTML.
  10. Exportar CV a PDF.
  11. Crear módulo de postulaciones.
  12. Crear recomendación salarial.
  13. Crear preparación de entrevistas.
  14. Agregar scraping o integraciones externas permitidas.
  15. Agregar Celery para tareas en segundo plano.

11. MVP recomendado

Para no hacer el proyecto demasiado grande desde el inicio, el MVP debería enfocarse en:

  • Registro de perfil profesional del usuario.
  • Carga manual o importación simple de vacantes.
  • Matching usuario-vacante.
  • Generación de CV adaptado.
  • Recomendación salarial básica.
  • Gestión de postulaciones tipo CRM.
  • Preparación de entrevista con Groq.

El scraping avanzado y la postulación automática deberían dejarse para una fase posterior, cuando ya esté validada la utilidad principal del producto.

12. Conclusión

El backend de HirePath debe diseñarse como una plataforma de empleabilidad centrada en el candidato. La base de datos propuesta permite construir un sistema ordenado, trazable y escalable, donde cada decisión importante queda registrada: qué vacante se encontró, por qué se recomendó, qué salario se sugirió, qué CV se generó y qué respuestas se prepararon.

La clave técnica del proyecto será separar correctamente los datos reales del usuario, los datos externos de las vacantes y los contenidos generados por IA. Esa separación permitirá mantener confianza, trazabilidad y control humano durante todo el proceso de búsqueda laboral.
